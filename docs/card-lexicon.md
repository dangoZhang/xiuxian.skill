# 修炼卡术语对照表

这份表专门服务卡片文案。

原则只有三条：

- 先把话说清楚，再谈修仙味。
- AI 术语优先用常见中文写法，只有模型名、tokens、Agent 这类保留英文更自然时才保留英文。
- 用常见修仙小说语汇，不乱造体系。

## 卡片区块用语

| 卡片区块 | 建议标题 | 文案要求 |
| --- | --- | --- |
| 顶部标题 | 修仙.skil | 固定写法 |
| 顶部标语 | 蒸馏你的vibecoding能力 | 固定写法 |
| 主视觉 | 境界 \| 等级 | 只突出这两项 |
| 能力区 | vibecoding能力 | 要把能力讲全：这一层能做什么、这轮亮点、眼下短板 |
| 判词区 | 判词 | 只写“你现在在哪一层，这一层通常意味着什么水平” |
| 指引区 | 突破方向 | 重点写“下一轮该补什么” |
| 信息栏 | 法器 \| tokens | 只保留平台 / 模型和 token |
| 页脚 | 称呼 \| 时间 | 信息要短 |

## 修仙术语 / AI 术语映射

| 修仙术语 | AI 术语 | 适用场景 |
| --- | --- | --- |
| 炼丹 | 提炼 / 蒸馏 | 提炼方法、总结套路、压缩经验 |
| 炼化 | 复用 / skill 化 | 把重复方法做成 skill / 模板 / 可复用法门 |
| 法门 | 工作流 | 一套稳定可重复的协作流程 |
| 章法 | 系统打法 | 更上层的组织方式与整体打法 |
| 分身 | Agent | 代为执行的一具或多具智能体 |
| 驭器 | 工具调用 | 调用工具、命令、搜索、读写文件 |
| 起手法诀 | prompt 调整 | 起手问法、引导方向 |
| 回炉 | 迭代修正 | 继续改 prompt、继续改要求 |
| 收功 | 结果验证 | 对结果做检查、测试、验收 |
| 回验 | 回看确认 | 回看改动、确认结果成立 |
| 观气 | 上下文阅读 | 看卷宗、看上下文、看环境来路 |
| 气脉 | 上下文承接 | 长回合续住主线 |
| 转圜 | 补救适配 | 遇阻后换打法、缩范围、补漏 |
| 行功 | 执行推进 | 真正开始做事、推进实现 |
| 入世 | 真实项目 | 进入真实场景、真实业务回路 |
| 回流 | 反馈回路 | 从真实结果再回炼到下一轮 |
| 传法 | 团队复制 | 把方法复制给团队或客户 |
| 法器 | 平台 / 模型 | 当前主用平台或模型 |
| 灵石 | tokens | 累计消耗的 token |

## 推荐文案骨架

| 区块 | 推荐写法 |
| --- | --- |
| vibecoding能力 | 已经会把常见来回沉淀成工作流，熟悉任务通常能稳定推进到多步完成。这轮亮点在协作节奏和补救适配。最该补的是结果验证和回看确认。 |
| vibecoding能力 | 会把重复打法炼成可复用套路，同类问题不必每次从零开始。亮点是起手清楚，短板是验证留痕还不够。 |
| 判词 | 当前落在金丹 / L4。 |
| 判词 | 这一层已经会把常见来回沉淀成工作流，熟悉任务通常能稳定推进到多步完成。 |
| 突破方向 | 下一轮先把目标、边界和验收写在前面，再让 AI 动手。 |
| 突破方向 | 每轮结束都补一句：改了什么、怎么验证、还有什么没验。 |
