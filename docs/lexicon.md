# AI / vibecoding / 修仙词汇表

这份词表只服务一件事：

把 `vibecoding`、`Agent`、`LLM` 这些现代概念，翻成修仙叙事时依然顺口、依然能懂。

原则只有三条：

- 默认先说人话，再上修仙比喻
- 修仙说法不能压过技术含义
- 首次翻译如果生硬，就继续改写到像常见修仙小说语气

## 为什么这样写

过去很多“AI × 修仙”表达的问题，不是词不够多，而是词太直译。

例如：

- `tool use` 直译成“御器”会很中二，但别人未必知道你在说什么
- `prompting` 直译成“法诀”也许有味道，但第一次看到的人很难立刻懂
- `workflow` 直译成“功法”容易显得虚，不像真正的工程能力

所以这份表分三列：

- 原始 AI 概念
- 第一层直译
- 改写后更适合修仙小说语气的说法

## 默认优先保留的词

下面这些词，就算切到修仙模式，也建议保留现代写法：

- `vibecoding`
- `prompt`
- `Agent`
- `workflow`
- `tools`
- `verification`
- `tokens`
- 具体模型名，如 `gpt-5`、`Claude Sonnet`、`Codex`

原因很简单：这些词已经是今天 AI 协作里的“通用语”。全翻掉，技术味会变弱，可信度也会掉。

## 境界描述写法

| 等级 | 建议写法 | 对应含义 |
| --- | --- | --- |
| L1 | 还在试手 | 主要停留在单轮提问 |
| L2 | 初通气感 | 开始知道问法会改结果 |
| L3 | 炼成小术 | 能把小任务做成 |
| L4 | 根基渐稳 | 常见任务能稳定多步推进 |
| L5 | 丹法初成 | 开始有复用套路 |
| L6 | 可托分身 | 会先让 AI 走一段 |
| L7 | 分身成阵 | 会并行调动多 Agent 与工具 |
| L8 | 章法自立 | 开始搭流程和方法 |
| L9 | 入世磨法 | 能进真实项目持续修正 |
| L10 | 可传一脉 | 方法能稳定复制给团队 |

## 常见句式模板

### 默认人话版

- 你现在大概在 `L4`，已经能把常见任务稳定推进到多步完成。
- 这轮最强的是协作节奏，最弱的是结果验证。
- 下一轮最该补的是收尾动作，不要停在“改完了”，要停在“证实了”。

### 修仙版

- 你如今大概落在 `筑基 / L4`，根基渐稳，常见差事已能接连推进。
- 这一轮气机最顺的地方，在于你和分身往返衔接不断。
- 真正拖后腿的，不是出手，而是收功；若不验功留痕，修为就容易浮。

## 禁用写法

- 为了修仙而修仙的乱造设定
- 一句话里堆太多括号术语
- 把每个 AI 词都强行翻掉
- 读起来像游戏文案，不像修仙小说，也不像真实工程总结

## 灵感来源

这份表的底层参考了两边：

- 近一年的 `vibe coding`、`agentic coding`、`tool use`、`verification loop` 这些常见 AI 协作语汇
- 修仙小说里最稳定的一批叙事词，如境界、闭关、破境、识海、分身、法器、传法、出关

最终落地时，再以“读起来顺不顺”为最后一轮筛选标准。
